## 那晚我什么都没写，却"写"了一整个 App

凌晨两点。光标在空白的编辑器里闪烁了二十分钟。

按照过去的习惯，我会先搭项目结构、配置 Webpack、写路由、连数据库。但那天晚上我累了——我在 ChatGPT 的输入框里随手打了一句话：

> "帮我做一个番茄钟，界面要干净，支持自定义时长，数据存 localhost"

三十秒后，一个完整的 HTML + CSS + JS 代码块出现在我面前。

---

## 不是"生成代码"，是"对话式编程"

那一刻我突然意识到：我做的事情不是让 AI 替我写代码，而是在跟我的 _编程意图_ 进行对话。

传统的编程流程是 **需求 → 设计 → 实现 → 测试 → 部署**，每一步都精确、线性、可控。

Vibecoding 的流程变成了 **描述感觉 → 观察结果 → 迭代对话 → 逐步收敛**。你不需要在每个阶段都"知道自己在做什么"，你只需要持续地表达你想要什么。

这有点像即兴爵士乐——你知道你在哪个调上，但你不知道下一个音符会是什么。

---

## 三个让我真正信服的瞬间

### 1. 用自然语言 Debug

一个 CSS 动画的缓动曲线总是差那么一点感觉。以前我会打开 Chrome DevTools，手动调整 cubic-bezier 参数。

这次我直接说："这个动画的结束动作太硬了，像刹车而不是滑行。"

AI 给了我 `cubic-bezier(0.68, -0.55, 0.27, 1.55)`。

行云流水。

### 2. 重构一整个服务只用了一句话

我有一个用 Express 写的 API 服务，想迁移到 Hono + Cloudflare Workers。

我把整个 index.ts 粘贴过去，说："把这个改成 Hono 框架，部署到 Cloudflare Workers 上。"

30 秒后，所有中间件、路由、错误处理全部改好了，包括 wrangler.toml 配置。

### 3. 做界面时不需要"设计稿"

我在做一个个人项目的登录页。没有设计稿，没有 Figma。我只说了：

> "做一个登录页，风格参考早期 Apple 官网——大图、大量留白、Helvetica 字体、只有一个 CTA。"

出来的结果比我自己手写 CSS 好看一百倍。

## Vibecoding 不是"无脑编程"

批评者会说：这根本不是编程，只是复制粘贴。

但他们没注意到的是——_意图的表达_ 本身就是编程的核心技能。你需要：
- 清晰描述你想要什么
- 判断结果是否符合你的预期
- 在对话中逐步引导向正确的方向
- 知道什么时候推倒重来

这比记忆 API 文档更需要品味和判断力。

---

## 现在我怎么工作

我现在的工作方式变成了三阶段：

1. **Vibe 阶段**：用自然语言描述意图，获取第一版代码
2. **Review 阶段**：仔细审查生成代码的逻辑、安全性、边界情况
3. **Polish 阶段**：迭代调整细节，直到代码"感觉对"

第一阶段可能只占 20% 的时间，但它决定了整个项目的骨架和气质。

---

## 给想尝试的人的建议

- **先相信，再验证**：不要一开始就对 AI 的输出吹毛求疵，先让它跑起来
- **一次只改一件事**：迭代比一次性需求更有效
- **说"不要什么"比说"要什么"更有用**：负面约束往往更精确
- **把 AI 当结对编程的伙伴**，而不是自动化工具
