## 前言

过去半年我几乎完全依赖 vibecoding 工作流完成了 4 个项目。在这个过程中逐渐磨合出了一套相对稳定的工具链。

这篇文章不是"工具列表"，而是我自己每天在用的东西——有踩过的坑，也有意外发现的宝藏。

---

## 第一层：对话引擎

这是 vibecoding 的心脏。选对对话引擎直接决定了你的效率天花板。

### Claude（主力）

**为什么选它**：代码质量和能力是所有模型中最好的。尤其是长上下文能力，让它成为 vibecoding 的首选。

**最佳实践**：
- 使用 Projects 功能把项目规范文档预先上传
- 把 package.json / tsconfig.json 粘贴进对话，让模型理解项目环境
- 每次会话聚焦一个模块，避免上下文过长导致质量下降

**局限**：单次消息长度有上限，超大文件需要拆分处理。速率限制在高峰时段会比较严格。

### ChatGPT o1-preview（攻坚）

**适合场景**：遇到需要深度推理的复杂逻辑——算法设计、Bug 根因分析、架构决策。

**使用时机**：不要用 o1 做日常编码（它太慢了），只在需要"聪明"而非"快"的时候打开。

### Cursor Composer（编辑实时）

**定位**：编辑器内的 vibecoding，适合边写边改的流式工作。

**技巧**：用 `@Files` 引用上下文文件，用 `@Docs` 引用框架文档，质量比纯 prompt 高一个量级。

## 第二层：前端快速搭建

### v0.dev

**一句话**：把你口述的 UI 描述变成 React + Tailwind 代码。

**最佳用法**：
- 先描述整体布局 → 再逐个组件微调
- 用截图作为参考输入，比文字描述准确 10 倍
- 预览确认后再复制代码

**陷阱**：默认的配色倾向紫蓝渐变（土味 AI 风格），记得后面手动调整成自己的设计系统。

### Bolt.new

**不同于 v0**：它不只是 UI，而是全栈——可以在浏览器里直接跑一个 Next.js 项目。

**最适合**：快速搭建 MVP，前后端一起出。

## 第三层：后端与部署

### Replit Agent

**场景**：需要快速搭建一个带后端的完整应用，但不想管 DevOps。

**体验**：描述需求以后它会自己规划、写代码、运行、调试，你就像一个跟着施工现场走的项目经理。

**注意**：复杂逻辑需要手动介入，Agent 对边缘场景的覆盖不够。

### Vercel + AI SDK

**我的部署标配**：
- GitHub 推代码 → Vercel 自动部署
- AI SDK 用来做流式响应
- Analytics 和 Speed Insights 开箱即用

一条 `git push` 完成从代码到线上。

---

## 第四层：Style Generation

### Midjourney → 代码参考

**用法**：不是生成"好看的图"，而是生成"设计参考"。

流程：描述你想要的风格 → 获取视觉参考 → 告诉 AI "按照这个风格写 CSS"。

我曾经用一张 Midjourney 生成的手绘风格示意图，让 Claude 写出了一个博客的完整 CSS——字间距、留白比例、配色方案全对。

---

## 我的日常工作流

1. **构思阶段**：手动画草图或写一段描述文字
2. **原型阶段**：v0.dev / Bolt.new 快速出 MVP
3. **开发阶段**：Cursor + Claude 组合，迭代推进
4. **部署阶段**：git push → Vercel 自动上线
5. **迭代阶段**：回到步骤 3，局部优化

这个流程的核心不是"用不用 AI"，而是"什么时候用哪个 AI"。

---

## 最后的提醒

工具会过时。列表会变化。

但底层原则不会变：

> **用自然语言精确表达意图的能力，是 vibecoding 时代最核心的技能。**

培养你的"描述力"——比学习任何一个具体工具都重要。
